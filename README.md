## üìñ About The Project

An elegant, interactive web application that teaches the **SOLID principles** of object-oriented programming through practical React and TypeScript examples. Each principle is explained with clear definitions, real-world code examples (both good and bad practices), and beautiful UI animations.

### ‚ú® What are SOLID Principles?

SOLID is an acronym for five design principles that make software designs more understandable, flexible, and maintainable:

- **S** - Single Responsibility Principle
- **O** - Open/Closed Principle
- **L** - Liskov Substitution Principle
- **I** - Interface Segregation Principle
- **D** - Dependency Inversion Principle

## üì¶ Installation

### Prerequisites

- Node.js 18+ or Bun
- npm, yarn, pnpm, or bun

### Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/mohamedkbx/React-SOLID-Principles.git
   cd React-SOLID-Principles
   ```

2. **Install dependencies**
   ```bash
   npm install
   # or
   yarn install
   # or
   pnpm install
   # or
   bun install
   ```

3. **Start the development server**
   ```bash
   npm run dev
   # or
   yarn dev
   # or
   pnpm dev
   # or
   bun run dev
   ```

4. **Open your browser**
   
   Navigate to `http://localhost:5173`

## üé® SOLID Principles Covered

### 1Ô∏è‚É£ Single Responsibility Principle (SRP)
> A class should have only one reason to change.

Learn how to create focused, maintainable components that do one thing well.

### 2Ô∏è‚É£ Open/Closed Principle (OCP)
> Software entities should be open for extension but closed for modification.

Discover how to design flexible systems that can grow without breaking existing code.

### 3Ô∏è‚É£ Liskov Substitution Principle (LSP)
> Objects should be replaceable with instances of their subtypes without altering correctness.

Understand proper inheritance and interface implementation.

### 4Ô∏è‚É£ Interface Segregation Principle (ISP)
> No client should be forced to depend on methods it does not use.

Learn to create lean, focused interfaces instead of bloated ones.

### 5Ô∏è‚É£ Dependency Inversion Principle (DIP)
> Depend on abstractions, not concretions.

Master dependency injection and inversion of control.
